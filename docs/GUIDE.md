## 1. Что такое JentleMemes Launcher

JentleMemes Launcher — это десктопный лаунчер для Minecraft: Java Edition,
написанный на **Tauri 2 (Rust + Svelte 4)**. Он совмещает в себе:

- собственное ядро запуска **FluxCore v3** с CoilCache и снапшотами цепочек профилей;
- встроенный магазин модов (Modrinth + CurseForge) с фильтрацией по версии/загрузчику;
- полноценную социальную часть (друзья, чат, голос) через сервер `jentlememes.ru`;
- гибкую систему оформления из **трёх независимых осей**: визуальный пресет,
  светлая/тёмная тема, акцентная палитра;
- оверлей поверх игры с управлением сессией и хвостом логов;
- поддержку Linux (X11 + Wayland) и Windows; macOS — вспомогательная, не все функции.

Ключевой показатель: **холодный запуск Minecraft 1.20.x (Forge) на среднем ноуте ~13 секунд**
(Prism Launcher на той же машине — ~14 с; см. `CHANGELOG` и `docs/LAUNCH_INTERNALS.md` —
там разобран весь пайплайн).

## 2. Установка

### 2.1. Linux (AppImage / .deb / .rpm)

**Arch Linux / CachyOS**

```bash
# из AUR (рекомендуется)
yay -S jentlememes-launcher-bin
# или вручную
curl -LO https://jentlememes.ru/dl/linux/jentlememes-launcher_2.0.0_amd64.AppImage
chmod +x jentlememes-launcher_2.0.0_amd64.AppImage
./jentlememes-launcher_2.0.0_amd64.AppImage
```

**Debian / Ubuntu / Pop!_OS**

```bash
wget https://jentlememes.ru/dl/linux/jentlememes-launcher_2.0.0_amd64.deb
sudo apt install ./jentlememes-launcher_2.0.0_amd64.deb
```

**Fedora / openSUSE**

```bash
sudo rpm -i jentlememes-launcher-2.0.0.x86_64.rpm
```

**Системные зависимости** (любой дистр, если собираете сами):

- `webkit2gtk-4.1` (жёсткий пин на `2.0.2`, см. `src-tauri/Cargo.toml`);
- `libssl` 1.1 или 3;
- опционально: `grim` или `maim` или `scrot` — для скриншотов из оверлея;
- Java 17 / 21 / 25 — лаунчер может скачать сам, но системная ускоряет первый запуск.

### 2.2. Windows (installer / portable)

- Installer (MSI): `jentlememes-launcher-2.0.0-x64-setup.msi` — ставится в `Program Files`,
  создаёт ярлык, автоматически регистрирует URL-хендлер для OAuth.
- Portable: `jentlememes-launcher-2.0.0-x64-portable.zip` — распаковать куда угодно,
  запустить `jentlememes-launcher.exe`. Данные ложатся в `%APPDATA%\jentlememes-launcher`
  (если не переопределено через флаг `--data-dir`).

**Требования**: Windows 10 1809+ / Windows 11. WebView2 Runtime подтягивается
автоматически.

### 2.3. Сборка из исходников

```bash
git clone https://github.com/jentlememes/launcher
cd launcher
npm install
cd src-tauri && cargo fetch && cd ..
npm run tauri dev       # dev-режим
npm run tauri build     # релиз; артефакты в src-tauri/target/release/bundle/
```

Нужно: Node 20+, Rust 1.78+, Tauri CLI (`cargo install tauri-cli`).

## 3. Первый запуск и Onboarding

При первом старте вы увидите **сплеш-экран** с анимированным логотипом и чек-листом
фаз:

1. **Настройки** — загружается/создаётся `settings.json`.
2. **Onboarding** — проверка, проходил ли пользователь мастер первичной настройки.
3. **Сессии** — мягкий рефреш Microsoft-токенов, если они есть в кеше.
4. **Профиль** — загрузка активного аккаунта.

После сплеша (минимум 650 мс, чтобы не мигал) запустится **OnboardingWizard**.

Мастер состоит из 9 шагов:

| Шаг | Назначение |
| --- | --- |
| Приветствие | краткое объяснение, что настроим |
| Тема | акцентная палитра: `jentle-green`, `purple`, `red`, `gold`, `cyan`, `furry-pink`, кастомный HEX |
| Визуал | визуальный пресет: `blend`, `modrinth`, `discord`, `legacy`, `glass` |
| Панель | сайдбар слева/справа, компактный/полный, либо вкладки сверху/снизу |
| Масштаб | `--ui-scale` от 90% до 125% |
| Модалки | стиль всплывающих окон: minimal/glass/dense/sheet |
| Кнопки / HUD | где показывать виджет загрузок (углы или скрыть) |
| Анимации | полный режим или «Спокойный» (`reduce-motion`) |
| Готово | сохранение и выход |

Мастер можно пропустить (кнопка «Пропустить») — все значения останутся по умолчанию.
Повторно открыть мастер: **Настройки → Расширенные → «Снова показать onboarding»**
(или ручной сброс через `FORCE_ONBOARDING=1` при старте).

## 4. Главное окно и навигация

### 4.1. Боковая панель

По умолчанию — слева, «expanded» (иконка + подпись). Секции:

- **Главная** (`home`) — дашборд: текущая игра, активный аккаунт, новости.
- **Сборки** (`library`) — список инстансов с иконками, запуск, настройки сборки.
- **Обзор** (`discover`) — поиск модов/модпаков/ресурс-паков на Modrinth и CurseForge.
- **Скины** (`skins`) — 3D-превью скина, загрузка кастомного, смена плаща (для
  премиум-аккаунтов).
- **Новости** (`news`) — агрегатор блогов лаунчера, Mojang, FluxCore.
- **Аккаунт** (`account`) — управление профилями.
- **Настройки** (`settings`) — все параметры.

Быстрое переключение плотности панели: клик по иконке «свёрнуть» в верхней части
сайдбара, либо Ctrl+B.

### 4.2. Палитра команд (Ctrl+K)

Самый быстрый способ делать всё в лаунчере 2.0 — глобальная палитра команд.
Вызов: **Ctrl+K** (на macOS — Cmd+K), либо клик по кнопке «Quick Search» в
заголовке.

В палитре работает нечёткий поиск (`fuse.js`) по заголовкам, описаниям и
ключевым словам. Примеры запросов:

- «главная» или `home` — перейти на дашборд;
- «настройки сеть» — открыть Настройки → Сеть;
- «новая сборка» — создать инстанс;
- «modrinth cobblemon» — найти модпак в Обзоре;
- «папка данных» — открыть `~/.local/share/jentlememes-launcher/` в файловом
  менеджере;
- «тема purple» — сменить акцент на фиолетовый прямо из палитры.

Любой таб может регистрировать свои команды через `registerCommands()` из
`src/lib/commandRegistry.ts` — при размонтировании таб обязан снять их.

### 4.3. Горячие клавиши

| Комбинация | Действие |
| --- | --- |
| Ctrl+K | открыть палитру команд |
| Ctrl+Shift+P | то же самое (alias) |
| Ctrl+B | свернуть / развернуть сайдбар |
| Ctrl+L | фокус на поиск в Discover |
| Ctrl+, | открыть Настройки |
| Alt+\` | открыть/закрыть игровой оверлей (настраивается) |
| F5 | обновить активный таб |
| Esc | закрыть модалку / палитру |

## 5. Аккаунты

### 5.1. Microsoft (премиум)

1. **Аккаунт → Добавить Microsoft**.
2. Откроется встроенный браузер с `login.live.com` → `xbl.xboxlive.com` →
   `api.minecraftservices.com`.
3. После успеха лаунчер сохранит `access_token`, `refresh_token`, UUID, name,
   ownership-флаг и тип скина в зашифрованном `profiles.json` внутри `data_dir`.

Лаунчер поддерживает **Microsoft Ownership Bypass** (опция в Настройках) —
позволяет играть с премиум-токеном, но без проверки на владение игрой. Это нужно
только для пользователей с ограниченным регионом/платежом; используйте честный
аккаунт, когда можете.

### 5.2. Offline / cracked

**Аккаунт → Добавить Offline**. Вводите никнейм — получите UUID `OfflinePlayer:<name>`
(детерминированный hash). Работает только на серверах с `online-mode=false`.

### 5.3. Авто-обновление токена

До 2.0 лаунчер делал 5 HTTP-запросов перед КАЖДЫМ запуском игры, даже если токен
валиден — это давало +10–12 с к холодному старту.

В 2.0 появился **локальный JWT-expiry check** (функция `ms_token_expired`):

```
refresh_account_for_launch:
  1. если токен не истёк → сразу OK (без сети)
  2. иначе:
      a) refresh ms_token через live.com
      b) xbl_authenticate
      c) xsts_authorize
      d) mc_login_with_xbox
      e) fetch_mc_profile
```

Кроме того, `prepare_launch` и `refresh_account_for_launch` теперь выполняются
**параллельно** (`Promise.allSettled` во фронте). Это сократило холодный старт с
22 с до 12.9 с (замеры из пользовательских логов).

### 5.4. Несколько профилей

Одновременно можно иметь сколько угодно Microsoft + Offline профилей. Активный
выбирается кликом в **Аккаунт → список справа**. Аватары кешируются в памяти,
баст-номер инвалидирует кеш при смене скина.

## 6. Сборки (инстансы)

### 6.1. Создание новой сборки

**Сборки → Новая**. Поля:

- **Название** — отображаемое имя.
- **Версия Minecraft** — список с фильтрами: release / snapshot / alpha-beta.
- **Загрузчик**:
  - `vanilla` — чистый MC;
  - `fabric` — мгновенно, поддерживает 1.14+;
  - `quilt` — fork fabric, поддерживает fabric-моды;
  - `forge` — классика, поддерживает большие модпаки;
  - `neoforge` — fork forge, активно развивается;
  - `liteloader` — только 1.12.2 и ниже.
- **Версия загрузчика** — подтягивается из нашей API (`get_specific_loader_versions`).
- **Иконка** — опционально, картинка 128×128.

### 6.2. Вариант: импорт модпака с Modrinth

**Обзор → Modrinth → ищете пак → Установить**.

Лаунчер скачает `.mrpack`, распакует `modrinth.index.json`, проверит SHA1 каждого
файла, докачает недостающие моды/ресурс-паки/конфиги и создаст готовую сборку с
правильным загрузчиком и версией MC.

Источник пака сохраняется в `pack_source.json` инстанса — потом лаунчер умеет
обновлять пак через ту же кнопку.

### 6.3. Вариант: импорт с CurseForge

Через **Обзор → CurseForge**. Нужен API-ключ (у нас зашит дефолтный; если
модпак приватный / платный — CF вернёт 403, это нормально).

### 6.4. Параметры сборки

**Сборки → иконка шестерёнки на карточке**. Вкладки:

- **Общее** — имя, иконка, команда после запуска (shell), версия MC, загрузчик.
- **Java и ОЗУ** — можно переопределить глобальные настройки. Если переопределения
  нет — используется `settings.java_home` и `settings.ram_mb`.
- **JVM-аргументы** — поле для дополнительных флагов (будут добавлены после
  стандартных G1/Log4j).
- **Окружение** — переменные среды для процесса MC (например
  `MESA_GL_VERSION_OVERRIDE=4.5`).
- **Запуск** — отдельный сервер для авто-подключения, аргументы MC.

### 6.5. Управление модами и ресурс-паками

**Сборки → карточка → «Файлы»** — открывает встроенный браузер `mods/`, `config/`,
`resourcepacks/`, `shaderpacks/`, `saves/`. Поддерживаются:

- drag-n-drop файлов из системы;
- копирование путей в буфер;
- открытие в системном файловом менеджере;
- быстрое отключение мода (добавляется `.disabled` к имени).

### 6.6. Удаление / переименование / ремонт

- **Переименовать**: ⋮ → «Переименовать» или из палитры команд.
- **Удалить**: ⋮ → «Удалить». Есть галочка «вернуть в корзину» (на Linux
  используется `trash-cli`, на Windows — `SHFileOperation`).
- **Ремонт**: ⋮ → «Починить ядро». Пересоздаёт цепочку профилей, очищает
  CoilCache, перекачивает ключевые jar-ы. Полезно, если выдаёт
  `ClassNotFoundException` или `NoSuchMethodError`.

## 7. Запуск игры

### 7.1. Что происходит под капотом

При нажатии **«Играть»** на карточке сборки:

1. Фронт параллельно стартует `prepare_launch(instance_id)` и
   `refresh_account_for_launch(account_id)`.
2. `prepare_launch` делает:
   - резолв цепочки профилей (vanilla → loader);
   - проверку CoilCache (хеш цепочки + mtime snapshot’а);
   - докачку недостающих libs и natives;
   - разархивацию natives в `<instance>/natives/`.
3. `refresh_account_for_launch` — локальная проверка expiry → при необходимости
   полный OAuth-refresh.
4. Когда обе задачи завершены, фронт зовёт `fluxcore_launch(instance_id)`.
5. Бэкенд собирает полный classpath, JVM-аргументы и запускает Java-процесс.
6. Первая строка stdout Java фиксирует метрику `first_line_emitted` (видно в
   логе лаунчера).

Подробности — в `docs/LAUNCH_INTERNALS.md`.

### 7.2. Тёплый и холодный запуск

- **Холодный** — все файлы с нуля (или сильно менялся модпак). 30–60 с.
- **Тёплый** — CoilCache попадает, JVM-process уже грет. На нашем тесте 1.20.1
  Forge — ~13 секунд от клика до окна Minecraft.

Ключевые флаги JVM, которые мы ставим всегда (не переопределяйте, если не
знаете, что делаете):

- `-XX:+UseG1GC`
- `-XX:+UnlockExperimentalVMOptions` (должен идти ПЕРЕД экспериментальными
  G1-флагами — отдельная история была с `G1NewSizePercent`)
- `-XX:G1NewSizePercent=20 -XX:G1ReservePercent=20 -XX:MaxGCPauseMillis=50 -XX:G1HeapRegionSize=32M`
- `-XX:+PerfDisableSharedMem`
- `-Dlog4j2.formatMsgNoLookups=true -Dlog4j.skipJansi=true`
- `-Dio.netty.tryReflectionSetAccessible=true`
- `-Duser.language=en -Djline.terminal.type=dumb`

### 7.3. Индикатор прогресса

Во время `prepare_launch` на карточке отображается полоса прогресса с тайминг-
разметкой:

```
[JentleMemes] ⏱ phase=init Δ=12ms total=12ms
[JentleMemes] ⏱ phase=resolve_chain Δ=94ms total=106ms
[JentleMemes] ⏱ phase=cache_check Δ=7ms total=113ms
[JentleMemes] ⏱ phase=java_and_args Δ=46ms total=159ms
[JentleMemes] ⏱ runner plan (digest+snapshot) Δ=58ms
```

Это живые события, летящие из Rust в фронт через `log_{instance_id}` event.

### 7.4. Просмотр логов

- **Главная → «Последний лог»** — показывает хвост `logs/latest.log` текущей
  сборки.
- **В оверлее** — кнопка «Логи» открывает live-tail с обновлением раз в 1.5 с и
  кнопкой «скопировать».
- **Полный файл**: `<data_dir>/instances/<id>/logs/latest.log` — Minecraft сам
  его ротирует через Log4j.

## 8. In-Game Overlay

### 8.1. Как вызвать

Оверлей — это отдельный Tauri-window, который ловит позицию окна Minecraft и
показывается поверх него. Открытие:

- горячей клавишей (Alt+\` по умолчанию, меняется в Настройки → Расширенные → «Ingame
  overlay hotkey»);
- программно через команду палитры («Открыть оверлей»).

### 8.2. Панель управления

В верхней строке оверлея (новое в 2.0):

| Кнопка | Действие |
| --- | --- |
| **Стоп** | убить процесс текущей сессии (`stop_game_from_overlay`). |
| **Скрин** | снять скриншот (на Linux: `grim` → `maim` → `scrot` → `import`; на Windows: PowerShell + `System.Drawing`). Путь к файлу: `<instance>/screenshots/launcher-<timestamp>.png`. |
| **Логи** | показать/скрыть панель хвоста `latest.log`. |
| Затемнение | ползунок прозрачности фона оверлея (0–100%). |
| Панели | открыть каталог виджетов. |
| ✕ | скрыть оверлей (не убить игру). |

### 8.3. Виджеты и drag-n-drop

Виджеты располагаются в адаптивной сетке. У каждого в заголовке:

- **хват** (иконка вертикальных точек) — удерживая LMB, можно перетащить виджет
  в новую позицию сетки; порядок сохранится в `localStorage`;
- **крестик** — удалить виджет.

Каталог виджетов (полный список — `src/lib/overlayDashboard.ts`):

`game_session`, `game_stats`, `clock_date`, `tips_rotation`, `overlay_hotkey`,
`chat_feed`, `launcher_meta`, `api_ping`, `friends_online`, `conversations_count`,
`screen_info`, `memory_bar`, `session_pids`, `rect_source_detail`,
`keyboard_hints`, `linux_gl_tip`, `random_mc_fact`, `poll_meta`,
`instance_list`, `jvm_ram_tip`, `network_status`, `clipboard_hint`,
`modrinth_tip`, `shader_tip`, `coordinates_joke`, `water_reminder`,
`breathing_hint`, `seed_idea`.

### 8.4. Хвост логов

Панель **Логи** тянет последние 80 строк `logs/latest.log` через Rust-команду
`tail_game_log`. Под капотом:

```rust
pub fn tail_game_log(instance_id: String, lines: Option<usize>) -> Result<String, String> {
    let n = lines.unwrap_or(120).clamp(1, 2000);
    // ...читает <instance>/logs/latest.log и возвращает последние N строк
}
```

Обновление каждые 1.5 с. Кнопка **Скопировать** кидает текст в clipboard
(использует `navigator.clipboard.writeText`).

### 8.5. Скриншоты

Мы не пытаемся хукать GL-контекст Minecraft — вместо этого вызываем системную
утилиту. Это менее инвазивно, но скриншот получается всего экрана / активного
окна (в зависимости от утилиты и окружения).

**Linux приоритет**: `grim` (Wayland) → `maim` (X11) → `scrot` (X11) → `import`
(ImageMagick). Если ничего нет — ошибка с понятным сообщением.

**Windows**: PowerShell + `System.Drawing.Bitmap` + `CopyFromScreen`. Работает
из коробки на Windows 10/11.

## 9. Оформление: три оси кастомизации

2.0 перешёл от монолитной темы к трём **независимым** осям. На `<html>`
выставляются:

- `data-preset="blend|modrinth|discord|legacy|glass"` — визуальный пресет;
- `class="theme-*"` — акцентная палитра (цвет кнопок, ссылок, полосок);
- `data-mode="dark|light|auto"` — режим (тёмный/светлый/системный).

Итого: 5 × 2 × 7 = **70 штатных сочетаний** (+ кастомный HEX = 105).

### 9.1. Визуальный пресет

**Настройки → Оформление → Визуальный стиль**. Пять плиток с превью:

- **Blend** _(по умолчанию)_ — гибрид Modrinth + Discord + Linear. Универсальный.
- **Modrinth** — просторный, большие радиусы (14px), мягкие тени, комфортный
  `--density: 1.1`.
- **Discord** — плотный, `--density: 0.9`, радиусы 6px, строгие тени.
- **Legacy 1.1** — реанимирует визуал 1.1.0: сильные тени, орбы-свечения, pulse-glow
  на активных пунктах сайдбара, sheen-эффект на первичных кнопках. Файл
  `src/styles/preset-legacy.css` содержит все специфичные правила.
- **Glass** — экспериментальный акриловый: `backdrop-filter: blur(16px)`,
  полупрозрачные панели. Требует GPU-акцелерации в WebView.

### 9.2. Режим: тёмный / светлый / авто

Переключается в **Настройки → Оформление → Режим**. В `auto` лаунчер слушает
`prefers-color-scheme` и переключается автоматически.

Светлая тема — **настоящая**, а не «подкрашенная тёмная». Все акцентные палитры
имеют светлую и тёмную вариации.

### 9.3. Акцентная палитра

Штатные: `jentle-green` (дефолт, `#86a886`), `purple` (`#9d6dff`), `red`,
`gold`, `cyan`, `furry-pink`. Плюс **custom** с HEX-пикером.

Кастомный HEX можно задать прямо в палитре команд: «тема custom #ff5733».

### 9.4. Масштаб UI

`--ui-scale` от 0.85 до 1.6. Влияет на размеры шрифтов, кнопок, иконок. Удобно
на 4K / HiDPI.

### 9.5. Фон и прозрачность

- Картинка фона: **Настройки → Оформление → Фон**. Поддерживаются jpg/png/webp.
- Прозрачность UI: ползунок «Прозрачность интерфейса» (если выбран пресет,
  поддерживающий это).
- В пресете `glass` прозрачность максимальная по умолчанию.

### 9.6. Anim / reduce-motion

В **Настройках → Оформление** есть тумблер **«Спокойный режим»** — ставит
`jm-reduce-motion` на `<html>`, что глушит все CSS-анимации (сплеш, оверлей,
переходы). Используется на слабых ПК и для людей с чувствительностью к
движению.

## 10. Настройки — по разделам

### 10.1. Java и ОЗУ

- **Java home** — путь к JDK. Лаунчер умеет авто-детектить через `list_detected_java_runtimes`
  (см. раздел 12). Если ничего не найдено, есть кнопка **«Скачать Java»** —
  качает Eclipse Temurin.
- **ОЗУ (Xmx)**: 1024–32768 МБ. Подсказка — не ставить больше
  `totalRAM - 2 ГБ`, иначе GC может дать длинные паузы.

### 10.2. JVM-аргументы

Текстовое поле. Будут добавлены ПОСЛЕ наших стандартных. Примеры полезных:

```
-Xss2m                         # больший стек для шейдеров/модов-физиков
-Dfml.readTimeout=60           # увеличить таймаут Forge
-Dfml.earlyprogresswindow=false  # если висит «Mojang logo»
```

### 10.3. Сеть

- **JentleMemes API base** — `https://jentlememes.ru` по умолчанию. Нужно для
  социальных функций и custom packs. Можно указать кастомный, если поднимаете
  свой сервер.
- **Modrinth mirror** — альтернативное зеркало для ускорения загрузок.
- **Proxy** — HTTP/SOCKS5. Применяется для `reqwest` клиента на бэке.

### 10.4. Обновления лаунчера

- **Канал**: stable / beta / nightly. По умолчанию stable.
- **Авто-проверка**: при старте. Если доступно обновление — в правом верхнем
  углу вылезет badge.
- **Скачивание**: через GitHub Releases или `jentlememes.ru/dl/`. Верификация
  SHA256.

### 10.5. Data directory

`<data_dir>` — корень всех файлов лаунчера. По умолчанию:

- Linux: `~/.local/share/jentlememes-launcher/`
- Windows: `%APPDATA%\jentlememes-launcher\`
- macOS: `~/Library/Application Support/jentlememes-launcher/`

Можно **переопределить** в **Настройки → Data directory → «Выбрать папку»**.
После подтверждения лаунчер перезапустится и будет использовать новую директорию.
Старые данные придётся переносить вручную — это не миграция.

### 10.6. Discord Rich Presence

Показывает текущую сборку и время игры в Discord. Работает через `discord-rich-presence` 1.x.
Если Discord не запущен — тихо отключается.

### 10.7. Экспериментальные опции

Раздел «только на свой страх и риск»:

- `experimental_jvm_flags` — набор доп. флагов (прокси, ZGC вместо G1 и т.п.);
- `legacy_log4j_patch` — применить патч для CVE-2021-44228 к старым версиям MC
  (1.17-);
- `force_wayland` / `force_x11` — принудительный backend на Linux;
- `enable_fluxcore_trace` — писать все события FluxCore в отдельный `flux-trace.log`.

## 11. Социальные функции

### 11.1. Чат друзей

Вкладка **Чат** (если в настройках включён `show_friends_chat_tab`). Нужен
аккаунт `jentlememes.ru`. Поддерживаются:

- список друзей с онлайн-статусом;
- private и group-чаты;
- вложения (картинки, файлы до 25 МБ);
- прямые текстовые сообщения с markdown-подмножеством.

### 11.2. Голосовые комнаты

WebRTC (mediasoup на сервере). Комнаты создаются в чате через «пригласить в
звонок». Работает на Linux (PipeWire + pulse-bridge), Windows (WASAPI), macOS
(ограниченно — mic only).

## 12. Обнаружение Java на системе

Команда `list_detected_java_runtimes` сканирует:

**Linux**: `/usr/lib/jvm/*/bin/java`, `$JAVA_HOME`, `~/.sdkman/candidates/java/*`,
`/opt/java/*`, `<data_dir>/java/<major>/bin/java`.

**Windows**: `C:\Program Files\Java\*`, `C:\Program Files\Eclipse Adoptium\*`,
`<data_dir>\java\<major>\bin\javaw.exe`, `%JAVA_HOME%`.

**macOS**: `/Library/Java/JavaVirtualMachines/*/Contents/Home/bin/java`,
`/opt/homebrew/opt/openjdk*`, `$JAVA_HOME`.

Для каждого пути запрашивается `-version`, выделяется major (8/11/17/21/25) и
vendor (Temurin/Liberica/Zulu/Oracle/OpenJDK/GraalVM).

Minecraft 1.18+ требует ≥ 17. 1.21+ — ≥ 21. Java 25 работает везде, где работал
21, при условии что Forge/NeoForge совместимы (на свежих версиях — да).

## 13. Типовые проблемы и решения

### 13.1. «Java не найдена»

Симптом: при запуске сборки говорит «Java major X не найдена».

1. Перейти **Настройки → Java** и проверить путь вручную.
2. Если нет — нажать **«Скачать Java X»**. Размер ~150–180 МБ.
3. Если сеть режется — скачать вручную и указать путь к `bin/java`.

### 13.2. «GLXBadFBConfig» на Linux

Симптом: Forge 1.12.2 падает сразу с GLX-ошибкой.

- Открыть Настройки сборки → «Использовать PRIME» → выключить (если у вас не
  NVIDIA Optimus).
- Либо обновить `mesa` до 24.x.
- Либо установить переменную среды `MESA_GL_VERSION_OVERRIDE=4.5` в окружении
  сборки.

### 13.3. «ClassNotFoundException» при запуске Forge

Чаще всего — кривой `forge-installer.jar` или битый classpath. Решение:

- **⋮ → Починить ядро** на карточке сборки.
- Если не помогло — вручную удалить `<instance>/.fluxcore-cache/` и перезапустить.

### 13.4. Долгий холодный старт

Если первая строка Java приходит > 20 с после клика:

1. Проверить `logs/launcher-*.log` — есть ли `⏱ phase=...` с большим Δ.
2. Обычно виноват сетевой шаг (`download_libraries` или refresh MS).
3. На слабом диске (HDD) — 30–40 с нормально.

### 13.5. Microsoft: требуется повторный вход

Если `refresh_token` протух (> 90 дней неактивности), лаунчер покажет тост
«Требуется повторный вход». Решение — заново добавить Microsoft аккаунт.

### 13.6. Минусы и отличия от Prism Launcher

| Функция | Prism | JentleMemes 2.0 |
| --- | --- | --- |
| Импорт модпаков CF | ✓ | ✓ |
| Импорт модпаков Modrinth | ✓ | ✓ |
| Кастомные темы | базовые | 3 оси × 5 пресетов |
| In-game overlay | ✗ | ✓ (виджеты, HUD, DnD, хвост логов) |
| Социальные функции | ✗ | ✓ (чат, голос) |
| Командная палитра | ✗ | ✓ (Ctrl+K) |
| Производительность запуска | ~14 с | ~13 с |
| Кросс-платформа | Linux/Win/macOS | Linux/Win, macOS частично |
| Русская документация | частично | полная |

## 14. Файловая структура data-dir

```
<data_dir>/
├─ settings.json          # глобальные настройки
├─ profiles.json          # аккаунты (access_token, refresh_token, uuid, name)
├─ chrome.json            # разметка сайдбара/модалок
├─ custom_packs.json      # собственный список паков
├─ instances/
│  └─ <uuid>/
│     ├─ instance.json    # конфиг сборки (name, version, loader, ram...)
│     ├─ pack_source.json # если пак импортирован из Modrinth/CF
│     ├─ mods/            # моды
│     ├─ config/          # конфиги модов
│     ├─ resourcepacks/
│     ├─ shaderpacks/
│     ├─ saves/
│     ├─ logs/
│     │  └─ latest.log    # текущий лог Minecraft
│     ├─ screenshots/     # сюда же летят скрины из оверлея
│     ├─ natives/         # распакованные natives jar’ы
│     └─ .fluxcore-cache/
│        ├─ chain.json    # snapshot цепочки профилей
│        └─ install.json  # sentinel-файл для download_game_files
├─ libraries/             # общие Maven-артефакты
├─ assets/                # ассеты MC (hash-split)
├─ versions/              # vanilla манифесты и jar’ы
├─ java/                  # скачанные JDK
│  ├─ 17/
│  ├─ 21/
│  └─ 25/
├─ logs/                  # логи самого лаунчера (ротируются ежедневно)
│  └─ launcher-2026-04-24.log
└─ backgrounds/           # пользовательские фоны
```

## 15. Конфиг-файлы

- `settings.json` — `src-tauri/src/config/mod.rs` → `LauncherSettings`. Главные
  поля: `theme`, `visual_preset`, `accent_palette`, `chrome_layout`, `ram_mb`,
  `java_home`, `ui_scale`, `reduce_motion`, `show_friends_chat_tab`,
  `jentlememes_api_base_url`, `discord_rpc_enabled` и десятки других. Все поля
  имеют `#[serde(default = ...)]`, так что битый JSON не ломает старт.
- `profiles.json` — шифруется `rsa` + машинный ключ.
- `instance.json` — на сборку. Структура — `InstanceConfig` / `InstanceSettings`.
- `chain.json` внутри `.fluxcore-cache/` — snapshot цепочки профилей с mtime и
  sha1 ключевых json’ов.

## 16. Телеметрия / приватность

Мы **не** собираем никаких данных о вас без явного согласия.

- Логи пишутся только локально в `<data_dir>/logs/`.
- Crash-репорты — только если вы нажали «Отправить» в модалке ошибки. Отправка
  идёт в `jentlememes.ru/api/v1/crash`, тело содержит: версию лаунчера, ОС,
  stack-trace, последние 200 строк лога. Без логина, без UUID.
- Анонимная метрика запуска — только если включили в Настройках
  («Help improve the launcher»). Отправляет `{version, os, startup_ms, loader, mc_version}`.
